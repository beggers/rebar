## Goal
- Remove the collection-replacement benchmark-anchor helper layer that still lives inline inside `tests/benchmarks/test_source_tree_combined_boundary_benchmarks.py` by moving that shared support and its focused contract checks into dedicated benchmark-support files, so the combined owner file stops carrying both the collection-replacement signature logic and the standard-benchmark assertions that consume it.

## Acceptance Criteria
- Add one bounded shared benchmark-support module at `tests/benchmarks/collection_replacement_benchmark_anchor_support.py` for the collection-replacement positional-indexlike and keyword anchor helpers that currently live inline in `tests/benchmarks/test_source_tree_combined_boundary_benchmarks.py`:
  - move `_is_encoded_indexlike_payload(...)`, `_collection_replacement_keyword_parameter_name(...)`, `_collection_replacement_parameter_payload(...)`, `_collection_replacement_has_expected_unexpected_keyword_error(...)`, `_module_workflow_positional_indexlike_correctness_case_signature(...)`, `_collection_replacement_positional_indexlike_workload_args(...)`, `_collection_replacement_positional_indexlike_workload_signature(...)`, `_is_collection_replacement_positional_indexlike_workload(...)`, `_collection_replacement_expected_keyword_field(...)`, `_collection_replacement_positional_keyword_field(...)`, `_collection_replacement_keyword_correctness_case_signature(...)`, `_collection_replacement_keyword_workload_args(...)`, `_collection_replacement_keyword_workload_signature(...)`, and `_is_collection_replacement_keyword_workload(...)` onto that module;
  - keep the helper surface ordinary Python support code that stays focused on the existing collection-replacement contracts across `benchmarks/workloads/collection_replacement_boundary.py`; and
  - do not introduce a new benchmark-definition abstraction, registry, or wrapper layer over the standard benchmark definitions.
- Add focused contract coverage at `tests/benchmarks/test_collection_replacement_benchmark_anchor_support.py` for the moved helper surface:
  - cover the positional-indexlike and keyword workload classification rules;
  - cover the current signature shape for the positional-indexlike and keyword anchor families, including the keyword-error path that keeps unexpected-keyword rows in scope; and
  - keep the support tests focused on the extracted helper behavior instead of re-copying the full owner-file standard-benchmark matrix.
- Update `tests/benchmarks/test_source_tree_combined_boundary_benchmarks.py` so it imports and uses the new support module instead of defining the collection-replacement helper layer inline:
  - keep the existing `collection-replacement-module-positional-indexlike` and `collection-replacement-keyword` entries in `STANDARD_BENCHMARK_DEFINITIONS` on the same `collection_replacement_boundary.py` manifest path with the same anchored case ids;
  - remove the moved helper definitions from this file once the new support module and support-test file fully cover them; and
  - leave the compiled-pattern collection-replacement and pattern-window helper families in the owner file unless the extraction proves they are already shared beyond this bounded slice in the current checkout.
